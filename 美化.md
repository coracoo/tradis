# UI/UX 全面美化分析与优化方案

## 1. 总体设计诊断 (Design Diagnosis)

基于对 `Overview`, `AppStore`,`AppDeploy`, `Compose`, `Images`, `Volumes`,`Networks`,`Navigation` 八个核心视图的分析，结合人类视觉感知原理与格式塔心理学，当前系统在采用 "Claymorphism" (黏土拟态) 风格的基础上，存在以下主要改进空间：

本次只针对  "Claymorphism" (黏土拟态) 风格优化，不针对 `SaaS Model` 以及 `复古/终端` 两种主题进行美化。

### 1.1 视觉层次与流 (Visual Hierarchy & Flow)
- **现状**: 页面多采用标准的 "Filter Bar + Table" 结构。虽然功能清晰，但在视觉流上较为平铺直叙，缺乏重点引导。
- **问题**:
  - **Compose 视图**: 嵌套表格 (Nested Tables) 导致视觉层级过深，用户视线在展开行时容易迷失，难以区分“项目级”与“容器级”操作。
  - **Images/Volumes 视图**: 列表项高度单一，操作按钮与状态信息的视觉权重过于接近，缺乏主次之分。
- **原理引用**: 根据**格式塔心理学的“封闭性”原则**，嵌套表格未能形成独立的视觉封闭区域，导致信息干扰。

### 1.2 间距系统与韵律 (Spacing & Rhythm)
- **现状**: `layout.css` 定义了基础间距，但在具体组件中存在硬编码数值 (如 `margin: 2px`, `padding: 4px`)。
- **问题**: 
  - 间距不一致导致视觉“呼吸感”不均匀。
  - Claymorphism 风格需要更大的负空间 (Negative Space) 来突显立体感，当前部分列表过于紧凑。
- **优化目标**: 建立以 **4px** 为基准的栅格系统 (4/8/16/24/32px)。

### 1.3 视觉一致性 (Visual Consistency)
- **图标**: 存在 `el-icon`、自定义 SVG、图片等多种形式混用。尺寸、线宽、圆角不统一。
- **圆角**: AppStore 图标、卡片圆角、按钮圆角之间缺乏比例关联。

## 2. 重点优化方案 (Key Optimization Initiatives)

### 2.1 Compose 视图：子层级列表卡片化 (Cardization of Nested Lists)
**目标**: 将 `Compose` 视图中展开的“容器列表”从表格形式改造为卡片阵列，提升层级区分度。

- **设计方案**:
  - **容器 (Container)**: 摒弃 `el-table`，改用 Flex/Grid 布局。
  - **卡片样式**: 使用“凹陷”或“低海拔”黏土效果，与父级“凸起”行形成对比。
  - **布局**:
    ```css
    .container-grid {
      display: grid;
      grid-template-columns: repeat(auto-fill, minmax(280px, 1fr));
      gap: 16px;
      padding: 16px;
      background: rgba(var(--clay-bg-rgb), 0.5); /* 略微加深背景 */
      border-radius: var(--radius-lg);
    }
    ```
  - **卡片内容**: 左侧状态指示灯 + 图标，中间名称/ID，右侧快捷操作 (Logs, Shell, Restart)。

### 2.2 图标系统标准化 (Icon System Standardization)
**目标**: 消除硬编码图标，建立统一视觉语言。

- **策略**:
  1. **库选型**: 统一使用 `Element Plus Icons` (基于 SVG)。
  2. **封装组件**: 创建 `<StatusIcon :status="..."/>` 和 `<ActionIcon :name="..."/>`。
  3. **硬编码替换**:
     - 检查 `Images.vue`, `Volumes.vue` 中的 `img` 标签或内联 SVG。
     - 替换为统一的 `<el-icon size="20">...</el-icon>` 配合标准配色。

### 2.3 App Store 图标圆角规范 (App Icon Standardization)
**目标**: 解决应用图标形状不一 (圆、方、异形) 带来的杂乱感。

- **规范**:
  - **容器**: 强制正方形容器，设定统一背景色 (如纯白或微灰)。
  - **圆角**: 采用 "Squircle" (超椭圆) 或标准圆角 `border-radius: 22%` (参照 iOS 标准)。
  - **边框**: 添加 1px 半透明内描边 (`inset border`) 处理白色图标在白色背景下的边界问题。
  - **CSS 实现**:
    ```css
    .app-icon-wrapper {
      width: 64px;
      height: 64px;
      border-radius: 16px; /* 或 22% */
      background: #fff;
      display: flex;
      align-items: center;
      justify-content: center;
      box-shadow: var(--shadow-clay-inner); /* 内阴影增加质感 */
      overflow: hidden;
    }
    .app-icon-wrapper img {
      width: 80%; /* 留白 */
      height: 80%;
      object-fit: contain;
    }
    ```

### 2.4 布局与间距微调 (Layout & Spacing Tuning)
- **全局变量定义 (`layout.css`)**:
  - 增加 `--spacing-xs: 4px`, `--spacing-sm: 8px`, `--spacing-md: 16px`, `--spacing-lg: 24px`, `--spacing-xl: 32px`。
- **组件调整**:
  - **Filter Bar**: 统一高度为 60px，内部元素垂直居中，左右 Padding 统一为 24px。
  - **表格行高**: 增加默认行高，使得文字与上下边框有更舒适的距离 (建议 `padding: 12px 0`)。

| 阶段 | 任务 | 涉及文件 | 优先级 |

## 4. 预期效果 (Expected Outcome)
- **视觉降噪**: 通过统一间距和卡片化，减少信息密度过高带来的压迫感。
- **操作直觉**: Compose 视图的层级更清晰，误操作率降低。
- **品牌质感**: 统一的 Claymorphism 风格与精致的图标细节，提升整体专业度。
